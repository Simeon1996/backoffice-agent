# Manila Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Manila — a no-backend Next.js demo of a back-office document agent that classifies, extracts, validates, and routes documents (invoice/PO/contract/claim/HR form), with a paper-desk aesthetic and inked rubber-stamp decisions.

**Architecture:** Typed mock corpus in `lib/data/` is the source of truth (each document carries structured content + extracted fields with confidence). A **pure** engine in `lib/engine/` classifies, validates against a rulebook, and routes to a stamped decision — all pure functions of `(document, rulebook)`. Documents render as real "paper" templates with `data-field` tags; an overlay measures real element rects to draw extraction boxes. Zustand holds read-playback + working rulebook + human decisions.

**Tech Stack:** Next.js 16, React 19, Tailwind v4, Zustand 5, motion 12, lucide-react, cmdk, geist + self-hosted serif, Vitest 4 + Testing Library.

## Global Constraints

- No backend, no network at runtime, no env vars, no persistence. In-memory only; hard refresh resets.
- Deterministic: seeded, identical output every run.
- Match sibling-app conventions (`ops-agent`, `security-agent`): `lib/data` + `lib/engine` + `lib/store.ts`; `components/{shell,ui,docs,<surface>}`; design tokens in `app/globals.css` via `@theme`.
- Unit tests (Vitest) cover the pure engine, data integrity, and store reducers. UI verified by build + render.
- Fictional company **Northwind**; product **Manila**. Everything fabricated.
- Visual identity: dark ink chrome `#1c1a17` + warm paper `#f4efe4`; stamp-red `#b23a2e` used ONLY for stamps; go-green `#2f7d52` for APPROVED. Serif display + Geist Sans UI + Geist Mono figures. Respect `prefers-reduced-motion`.
- Fixed vocab: doc types `invoice | purchase_order | contract | claim | hr_form`; rule status `pass | warn | fail`; routing `auto_approve | needs_review | escalate | reject`.

## Phase 1 — Types & corpus

### Task 1: Core types

**Files:** Create `lib/data/types.ts`.

**Produces:**
```ts
export type DocTypeId = "invoice" | "purchase_order" | "contract" | "claim" | "hr_form";
export type RuleStatus = "pass" | "warn" | "fail";
export type RoutingKind = "auto_approve" | "needs_review" | "escalate" | "reject";
export type DocStatus = "queued" | "auto_filed" | "needs_review" | "approved" | "rejected";

export interface FieldSchema { id: string; label: string; kind: "text" | "money" | "date" | "id" | "enum"; }
export interface RuleDef {
  id: string; docType: DocTypeId; name: string; explain: string;
  severity: "fail" | "warn";          // severity when it does not pass
  defaultEnabled: boolean;
  threshold?: number;                  // for amount-style rules ($)
}
export interface DocTypeDef { id: DocTypeId; label: string; blurb: string; fields: FieldSchema[]; }

export interface ExtractedField { id: string; value: string | number; confidence: number; } // 0..1
export interface Document {
  id: string;                          // e.g. "INV-4471"
  type: DocTypeId;
  title: string;
  sender: string;                      // Person id
  receivedAt: number;                  // virtual ms since DAY_START
  amount?: number;                     // headline money, if any
  fields: ExtractedField[];            // extracted values w/ confidence
  content: Record<string, unknown>;    // structured content for the template
  /** ground-truth facts the rules read (kept separate from extracted display). */
  facts: {
    poMatch?: boolean; lineItemsReconcile?: boolean; vendorApproved?: boolean;
    signaturePresent?: boolean; clausesPresent?: boolean; expired?: boolean;
    policyActive?: boolean; withinCoverage?: boolean; requiredComplete?: boolean;
    managerAssigned?: boolean; withinPerDiem?: boolean; withinBudget?: boolean;
  };
}
export interface Person { id: string; name: string; org: string; role: string; }
```

### Task 7: Validator

**Files:** Create `lib/engine/validate.ts`. Test: `lib/engine/__tests__/validate.test.ts`.

**Produces:**
```ts
export interface RuleResult { ruleId: string; name: string; status: RuleStatus; message: string; }
export function validateDoc(doc: Document, rulebook: Rulebook): RuleResult[];
```
For each enabled rule in `rulesForType(doc.type)`: evaluate against `doc.facts`/`doc.amount`. A rule passes when its fact is true (or amount ≤ threshold for threshold rules); otherwise its result is `fail` or `warn` per `RuleDef.severity`, with a plain message. Disabled rules are omitted. Pure.
- [ ] **Write failing tests:**
```ts
import { describe, it, expect } from "vitest";
import { validateDoc } from "../validate";
import { DEFAULT_RULEBOOK, cloneRulebook } from "../../data/rulebook";
import { documentById } from "../../data/documents";

describe("validateDoc", () => {
  it("fails the 3-way PO match on INV-4471", () => {
    const res = validateDoc(documentById("INV-4471")!, DEFAULT_RULEBOOK);
    const po = res.find((r) => r.ruleId === "inv_po_match")!;
    expect(po.status).toBe("fail");
  });
  it("omits a disabled rule", () => {
    const rb = cloneRulebook(DEFAULT_RULEBOOK);
    rb["inv_po_match"].enabled = false;
    const res = validateDoc(documentById("INV-4471")!, rb);
    expect(res.find((r) => r.ruleId === "inv_po_match")).toBeUndefined();
  });
  it("a clean invoice passes all its rules", () => {
    const res = validateDoc(documentById("INV-4460")!, DEFAULT_RULEBOOK);
    expect(res.every((r) => r.status === "pass")).toBe(true);
  });
});
```

### Task 8: Router

**Files:** Create `lib/engine/route.ts`. Test: `lib/engine/__tests__/route.test.ts`.

**Produces:**
```ts
export interface RoutingDecision { kind: RoutingKind; queue: string; rationale: string; stamp: string; }
export function routeDoc(doc: Document, results: RuleResult[], rulebook: Rulebook): RoutingDecision;
```
Rules: any `fail` → if the fail is a hard policy (PO match, coverage, signature) → `needs_review` (stamp "NEEDS REVIEW") or `reject` for coverage breaches/expired; amount over threshold with otherwise-clean → `escalate` (stamp "ROUTED") to a senior queue; all pass → `auto_approve` (stamp "AUTO-FILED"). `queue` is a human/role label. Pure; deterministic.
- [ ] **Write failing tests:**
```ts
import { describe, it, expect } from "vitest";
import { routeDoc } from "../route";
import { validateDoc } from "../validate";
import { DEFAULT_RULEBOOK, cloneRulebook } from "../../data/rulebook";
import { documentById } from "../../data/documents";

const decide = (id: string, rb = DEFAULT_RULEBOOK) => {
  const d = documentById(id)!;
  return routeDoc(d, validateDoc(d, rb), rb);
};

describe("routeDoc", () => {
  it("routes INV-4471 to review under the default rulebook", () => {
    expect(decide("INV-4471").kind).toBe("needs_review");

    const rb = cloneRulebook(DEFAULT_RULEBOOK);
    rb["inv_po_match"].enabled = false;
    rb["inv_threshold"].enabled = false;
    expect(decide("INV-4471", rb).kind).toBe("auto_approve");
  });
  it("auto-approves a clean invoice", () => {
    expect(decide("INV-4460").kind).toBe("auto_approve");
  });
});
```

### Task 9: Read sequence + metrics

**Files:** Create `lib/engine/read.ts`, `lib/engine/metrics.ts`. Tests: `lib/engine/__tests__/read.test.ts`, `metrics.test.ts`.

**Produces:**
```ts
// read.ts
export type ReadStep =
  | { kind: "classify" }
  | { kind: "field"; fieldId: string }
  | { kind: "validate" }
  | { kind: "route" };
export function readSequence(doc: Document): ReadStep[]; // classify, one per field, validate, route
export const TOTAL_FIELDS_FIRST = 1; // classify index

// metrics.ts
export interface Throughput { total: number; byType: { type: DocTypeId; count: number }[];
  stpRate: number; exceptions: number; dollarsProcessed: number; topExceptions: { reason: string; count: number }[]; }
export function throughput(docs: Document[], rulebook: Rulebook): Throughput;
```
`readSequence`: `[{classify}, ...fields, {validate}, {route}]`. `throughput`: STP rate = share of docs routing `auto_approve`; exceptions = docs routing review/escalate/reject; dollarsProcessed = sum of amounts; topExceptions = ranked fail-rule names.
- [ ] Tests: `readSequence(doc)` length === doc.fields.length + 3 and starts with classify, ends with route; `throughput(DOCUMENTS, DEFAULT_RULEBOOK)` stpRate in 0..1, total === DOCUMENTS.length, dollarsProcessed > 0.
